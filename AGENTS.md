# AGENTS.md — правила для AI-ассистентов

Этот файл — единый источник правил при работе с html-landing-template.

## Работа с шаблоном

1. **Никогда не меняй `tailwind.config.js` без обсуждения** — это дизайн-система, изменения каскадные.
2. **При добавлении новой секции копируй паттерн существующей:** фоновый блок (`absolute inset-0` с img + overlay-left) + контент (`relative z-10 flex ...`).
3. **Все новые секции должны использовать дизайн-токены** из `tailwind.config.js`, а не хардкод.
4. **Не добавляй CSS-файлы** — весь кастомный CSS пиши в `src/input.css` в `@layer components`.
5. **Не добавляй npm-пакеты** — стек фиксирован: Tailwind v3 + PostCSS + Vanilla JS.

## Соглашения

- **Цвета:** используй `ink`, `paper`, `mist`, `smoke`, `accent` — не `#000`, не `black`.
- **Кнопки:** используй класс `btn-primary` или `btn-sticky` из `@layer components`.
- **Оверлей:** `overlay-left` для градиента слева на фоновых изображениях.
- **Анимации:** `data-reveal` для появления при скролле (все вариации — в `input.css`).
- **Отзывы:** используй класс `card` для карточек.
- **Размеры экранов:** `min-h-[100dvh]` для fullscreen, `min-h-screen` для обычных.
- **Контейнер:** `max-w-mobile` для мобильного вида, не меняй без необходимости.
- **Изображения:** WebP или AVIF. Всегда с `loading="lazy"` (кроме hero — `eager`).

## Деплой на GitHub Pages

Workflow: `.github/workflows/deploy.yml`. Срабатывает на push в `main`.
Сборка и деплой идут через GitHub Actions — `dist/` не нужно коммитить.

## Процесс адаптации под новый лендинг

1. Прочитать `src/index.html` — найти все `FIXME` комментарии.
2. Если нужна новая цветовая схема — обсудить с пользователем, менять `tailwind.config.js`.
3. Если нужна дополнительная секция — скопировать существующий шаблон.
4. **Проверить `.github/workflows/deploy.yml`:** поправить ветку (не `main` → `master`?) и команду сборки, если `package.json` менялся.
5. После изменений запустить `bun run build` для проверки.
